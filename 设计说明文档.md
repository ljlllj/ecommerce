# 优品商城 - 网页设计课程大作业设计说明

## 一、网站主题及目标用户

### 主题
本网站是一个名为"优品商城"的小型电商展示平台，定位为精选好物推荐商城。网站涵盖了数码电子、家居生活、服饰配饰、箱包出行四大商品品类，共12件精选商品。通过模拟真实电商平台的核心功能，展示 HTML、CSS、JavaScript 的综合应用能力。

### 目标用户
- 追求品质生活的年轻消费者（18-35岁）
- 希望通过简单快捷的方式发现和购买精选好物的用户
- 对商品品质有一定要求，愿意为更好的体验支付合理溢价的消费群体

## 三、技术实现要点

### HTML 语义化
- 使用 HTML5 语义标签：`<header>`、`<nav>`、`<section>`、`<footer>`、`<form>`
- 页面结构清晰，符合 W3C 标准
- 所有页面通过 `<nav>` 实现统一导航

### CSS 布局
- 采用 CSS 自定义属性（CSS Variables）统一管理配色、间距、圆角等设计令牌
- 使用 CSS Grid 实现商品网格布局、页脚多列布局
- 使用 Flexbox 实现导航栏、筛选栏等一维布局
- 响应式设计：`@media (max-width: 768px)` 适配移动端
- 统一的设计语言：卡片阴影、悬停动效、过渡动画

### JavaScript 行为
- **商品数据管理**：使用独立 `data.js` 管理12件商品数据，包含id、名称、分类、价格、描述等信息
- **购物车系统**：基于 `localStorage` 实现购物车持久化存储，支持增删改查和数量管理
- **搜索与筛选**：实时搜索过滤（input事件）、分类下拉筛选、价格排序（升序/降序）
- **表单验证**：登录/注册/留言三个表单均包含完整的客户端验证（邮箱格式、密码长度、确认密码一致性等）
- **订单管理**：下单时生成唯一订单号，存储到 localStorage，订单页展示历史订单及状态
- **Toast 提示**：自定义轻提示组件，用于操作反馈
- **购物车徽标**：全局导航栏实时显示购物车商品数量

## 四、遇到的困难与解决方案

### 困难1：购物车数据持久化
**问题**：页面刷新后购物车数据丢失，用户体验差。
**解决方案**：使用 `localStorage` API 实现购物车数据的本地持久化存储。每次购物车操作（添加、删除、修改数量）后立即同步到 localStorage，页面加载时从 localStorage 读取数据。

### 困难2：多页面间数据共享
**问题**：购物车数据需要在首页、商品列表、购物车页等多个页面间共享。
**解决方案**：将所有购物车操作函数（getCart、saveCart、addToCart等）封装在公共的 `main.js` 中，每个页面引入该文件即可调用同一套逻辑。使用 `localStorage` 作为数据中介，所有页面读写同一份数据。

### 困难3：商品详情页的数据传递
**问题**：从商品列表页点击进入详情页，需要传递商品ID。
**解决方案**：使用 URL 查询参数（`?id=3`），详情页通过 `URLSearchParams` 解析参数，从商品数据数组中查找对应商品。

### 困难4：表单验证的用户体验
**问题**：只在提交时验证不够友好，用户需要多次提交才能看到所有错误。
**解决方案**：采用"提交时全量验证 + 输入时实时清除错误"的策略。提交时显示所有错误信息，用户修改输入时自动清除对应字段的错误状态，兼顾严谨性和体验。
